# 校园供应链与 IDS 创新点

## 1. 业务系统与安全系统解耦

校园供应链主站负责业务承载，IDS 独立部署、独立运行。这样做的优势是主站接口和页面继续迭代时，不会把安全能力一起拖乱，后续对接和维护成本更低。

## 2. 面向业务通信链路的安全防护

本项目保护的不是抽象的网站页面，而是校园物资与物流供应链中的核心通信过程，包括页面访问、接口调用、文件传输三类业务通信。异常请求进入业务系统后，可以被实时识别、拦截和留痕。

## 3. 检测、阻断、告警、留痕一体化闭环

系统不是只做检测或只做展示，而是形成了完整闭环：

- 发现异常请求
- 判断风险类型和风险分值
- 执行阻断或隔离
- 生成事件记录
- 触发邮件通知和后续处置

这样评委能够看到从攻击发起到防护生效的完整链路。

## 4. 文件传输安全审计能力

供应链文件上传链路已经接入独立 IDS 的上传审计能力。正常文件可以放行，恶意样本会被 AI 审计并隔离到沙箱，同时生成事件记录和分析结果，体现了文件传输安全防护能力。

## 5. 攻防演示链路清晰

渗透测试平台负责发起探测与攻击动作，校园供应链作为业务承载目标，独立 IDS 负责检测和处置。三者职责分工明确，便于现场展示“攻击发起、业务受压、IDS 识别、防护生效”的完整过程。

## 6. 可交付、可迁移、可复现实战演示

项目已经整理为可直接打包交付的结构，供应链、IDS 和启动脚本可以一起分发。启动方式统一，便于在不同机器、不同目录下解压后直接运行，适合比赛答辩和现场演示。
