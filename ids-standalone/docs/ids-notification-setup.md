# 独立 IDS 通知配置说明

## 1. 邮件通知怎么发

这套独立 IDS 走的是标准 `SMTP`。

意思很简单：

- IDS 后端自己负责发邮件
- 但它不是自己充当邮件服务器
- 它会去连接你提供的外部 SMTP 服务

所以大多数情况下，你**不需要自己搭建邮件服务器**。

## 2. 最省事的做法

直接用现成的邮箱 SMTP 服务：

- 企业邮箱
- 云邮箱
- 第三方 SMTP 服务

你只需要在通知配置页填这些字段：

- `SMTP Host`
- `SMTP Port`
- `Username`
- `Password` 或授权码
- `From`
- `To`

## 3. 端口怎么选

常见就两种：

- `465 + SSL`
- `587 + STARTTLS`

建议：

- 如果你的服务商文档写的是 SSL，就勾 `SSL`
- 如果写的是 STARTTLS，就勾 `STARTTLS`
- 一般不要两个同时开

现在系统保存时也会自动兜底：

- 如果你同时开了 `SSL` 和 `STARTTLS`
- 后端会自动优先按 `SSL` 收口

## 4. 密码填什么

不一定是邮箱登录密码。

很多邮箱会要求：

- SMTP 专用密码
- 应用专用密码
- 授权码

如果你的邮箱后台给了“客户端授权码”或“应用密码”，优先填那个。

## 5. 配置保存在哪里

通知配置会保存到独立 IDS 的运行状态目录里：

- [notification_settings.json](/D:/ids/ids-backend/runtime/state/notification_settings.json)

敏感字段不会在前端直接回显：

- 邮件密码
- Webhook Secret

前端只显示“是否已配置”。

## 6. 测试流程

推荐顺序：

1. 先填 SMTP 参数
2. 点击“保存配置”
3. 再点“发送测试”
4. 看页面里的“最近一次测试结果”

如果失败，重点看：

- `smtp_host` 是否写对
- 端口是否和加密模式匹配
- 用户名/授权码是否可用
- 发件地址是否被服务商允许
- 服务器是否能连外网 SMTP

## 7. 什么时候才需要自建邮件服务器

只有这几类场景才考虑自建：

- 你们内网完全不允许连外部 SMTP
- 你们有合规要求，必须邮件全链路自管
- 你们本来就有内部邮件网关

如果不是这几种情况，直接接现成 SMTP 最省事。
