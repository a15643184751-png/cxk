# 通信配置讲解稿

## 搭配总架构讲稿的衔接讲法

在完成第二阶段边界安全防线构建、第三阶段核心业务平台部署、第四阶段纵深主动防御体系搭建以及第五阶段全流程联调与闭环验证之后，我们进一步在软件通信层面补齐了 IDS 的通信接入治理能力。

老师们现在看到的，就是 IDS 的通信配置中心。这个模块的核心作用，不是单纯设置一个访问地址，而是定义外部业务通信应该如何先进入 IDS，再由 IDS 判断后转发到校园物资与物流供应链平台。

也就是说，在这套架构里，防火墙负责边界区域控制，蜜罐负责攻击诱捕与路径溯源，堡垒机负责权限与运维审计，而 IDS 通信配置中心负责把业务访问链路真正前移到通信入口位置，形成前置接入、前置识别、前置阻断的通信安全能力。

## 对着页面的讲法

### 页面上方总览区

这里展示的是当前通信接入的总体状态，包括已经纳管了多少入口、当前配置写入到了什么位置，以及系统生成的接入摘要。

这部分的作用，是帮助我们快速确认当前通信链路是否已经建立完成，也方便在更换设备、调整 IP 或切换网段时快速恢复环境。

### 默认接入

这里定义的是最基础、也是最关键的一条主通信链路。

- 默认入口端口，决定外部请求先从哪个 IDS 入口进入。
- 默认前端 IP 和端口，决定正常页面流量应该转发到哪台业务主机。
- 默认后端 IP 和端口，决定接口请求应该转发到哪台业务后端。

也就是说，这一块配置的是通信从哪里进入、最终流向哪里，是 IDS 通信接入治理的核心。

### 接入摘要

这一块是对当前通信配置结果的直观呈现。

系统会把默认入口、多站点入口和域名入口自动整理出来，让老师可以直接看到：

- 当前有哪些入口
- 每个入口最终回源到哪里
- 哪条链路对应哪个业务目标

从答辩展示角度来说，这一块可以把抽象的通信配置转化成老师容易理解的通信链路结果。

### 附加入口映射

这一块主要解决一台 IDS 同时管理多个站点、多个端口的问题。

比如主站点走默认入口，第二个站点可以单独映射到另一个入口端口。这样 IDS 虽然统一作为安全接入点，但不同业务仍然可以被独立分流、独立转发、独立留痕。

### 域名入口映射

这一块主要解决同端口多站点接入的问题。

如果多个业务共用一个入口端口，IDS 仍然可以根据请求里的域名信息判断当前通信请求应该被转发到哪个目标站点。

这说明我们的通信治理既可以按端口做，也可以按域名做，具备更高的灵活性。

### 通信域标识与链路组织

这一块更多是控制台编排层。

它主要负责链路名称、通信域编号、标签、轮转方式、展示模式和轨迹染色等表达方式，让 IDS 大屏、日志、轮转页和态势页能更清晰地展示通信链路。

这部分不会改变网关真实转发结果，但会显著增强控制台的展示效果和现场讲解表现力。

## 重点配置与展示配置

### 重点配置

这些是真正会影响 IDS 运行和网关转发的关键配置：

- 默认入口端口
- 默认前端 IP
- 默认前端端口
- 默认后端 IP
- 默认后端端口
- 附加入口映射里的入口端口
- 附加入口映射里的前端回源地址
- 附加入口映射里的后端回源地址
- 域名入口映射里的域名列表
- 域名入口映射里的前端回源地址
- 域名入口映射里的后端回源地址

### 可讲但不用深讲

- 默认站点名称
- 默认站点标识
- 附加入口映射里的站点名称和站点标识
- 域名入口映射里的站点名称和站点标识

这部分主要服务于链路识别、日志命名和接入摘要展示。

### 主要用于展示编排

- 通信域编号
- 链路模板
- 协同分组
- 路由编排
- 会话封装
- 频段标签
- 同步时钟
- 展示模式
- 会话轮转
- 轨迹染色
- 中继分组
- 自动轮转
- 弹窗播报
- 数据包影子副本
- 链路保活

这一组参数更适合在答辩时讲成“控制台通信编排能力”和“链路可视化表达能力”，不用讲成真实网络配置。

## 最适合现场总结的一句话

这页里真正决定通信怎么经过 IDS 的，是入口端口、目标 IP、目标端口、附加入口映射和域名入口映射；其余参数更多是为了让控制台更像一个完整的通信安全工作台，不影响网关真实放行与拦截。
